# Project Brief

Design a tiny greeting utility workflow and break the work into exactly three independent chunks.

The architect must recommend these exact chunk ids and titles:
- `greeting-copy` -> `Implement chunk greeting-copy`
- `greeting-format` -> `Implement chunk greeting-format`
- `greeting-proof` -> `Implement chunk greeting-proof`

Each chunk goal should explain one independent slice of the tiny greeting utility.

The implementation role does not build a real program in this test. It only proves that the orchestrator used the architect's authored breakdown correctly by creating one chunk output file per recommended split.
